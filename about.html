// hypertext markup language

About

HTML — HyperText Markup Language — is the foundational language of the web. Every page you've ever visited is built on it. It describes the structure and meaning of content through a system of elements and attributes.

It was created by Tim Berners-Lee in 1991 as a way to share documents across the early internet. What started as a handful of tags has grown into a mature specification maintained by the WHATWG and W3C.

How it works

HTML documents are plain text files. A browser reads the file, parses the tags, and renders a visual page. Tags wrap content to give it meaning — a <p> is a paragraph, an <h1> is a top-level heading, an <a> is a link.

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello, world.</h1>
    <p>This is a paragraph.</p>
  </body>
</html>

The <head> contains metadata — invisible to the user but important for the browser and search engines. The <body> holds everything that appears on screen.

Semantic markup

Modern HTML emphasizes semantics — using the right element for the right purpose. A <nav> marks navigation, an <article> marks self-contained content, a <footer> marks the bottom of a page or section. This matters for accessibility, SEO, and maintainability.

Structure should reflect meaning, not appearance. Style is CSS's job.

Some commonly used semantic elements:

<header> <nav> <main> <article> <section> <aside> <footer> <figure> <time> <mark>

Attributes

Attributes provide additional information about elements. They sit inside the opening tag as key-value pairs.

<a href="https://example.com" target="_blank" rel="noopener">
  visit example
</a>

<img src="photo.jpg" alt="a description of the image" width="800">

<input type="email" placeholder="your@email.com" required>

The alt attribute on images, for instance, isn't optional in practice — it's what screen readers announce to blind users, and what displays if the image fails to load.

HTML today

The current version is HTML Living Standard — a continuously updated spec rather than a versioned release. It introduced structural elements, native <video> and <audio>, the <canvas> drawing API, form validation, and much more.

HTML is intentionally simple. It doesn't have logic or loops — that's JavaScript's job. It doesn't control appearance — that's CSS. What it does, it does well: describe content, establish structure, connect documents through links.

The web is built on a language anyone can read, write, and view source on. That openness was intentional — and it still matters.
